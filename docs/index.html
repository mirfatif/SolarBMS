Solar and Battery Management System Using Arduino Nano

Project Objective

This project aims to develop a solar and battery power management system using an Arduino Nano. The system prioritizes solar energy during daytime (in SUB mode) to power an inverter and charge a battery, while intelligently switching to utility power (WAPDA) when necessary. The primary objective is to ensure continuous power availability while protecting the battery from overcharge (high charge quantity), over-feeding (high charging rate), over-discharge (high discharge quantity), and overload (high discharge rate) conditions.

System Overview

  • Primary Power Source: 4 x 300W (Voc = 24V, Isc = 15A) PV solar array via a PWM charge controller.
  • Backup Power Source: 220V 50Hz Utility grid (WAPDA).
  • Storage: 12V 200Ah lead-acid tubular battery, which stays at ≈12.9V when fully charged, with no load and no charging.
  • Load: 600W inverter powering AC appliances.

Operational Logic

  1. During daytime, power is drawn from solar (according to solar on/off time settings and sunlight availability) to run the inverter and charge the battery.
    • You can configure time (in minutes, before configured sunlight time) in which load is shifted from grid to solar if sunlight is sufficient to bear the load. Otherwise, load is silently shifted back to grid.
    • Similarly, at evening, load is kept on solar as long as sunlight is sufficient.
  2. If the inverter load exceeds solar supply and the battery begins to discharge, switch to grid, even during daytime.
  3. At night:
    • If grid is available, use it instead of draining the battery.
    • If grid is unavailable, use the battery to run the inverter as long as the battery state remains within safe operating limits.
  4. If sunlight is not sufficient to produce the minimum required current e.g. due to morning / evening or clouds, load is silently shifted to grid.
  5. If USB mode is selected (instead of SUB), grid is always given priority. This can be useful in winters when not much power is needed during daytime. So inverter runs only during power outages.

Battery Protection Rules

  • Over-discharge protection
    Inverter is turned off if battery voltage drops below BVlow, even if there is no grid. A 2 minutes window is given if voltages remain above BVcrit.
    Discharging current is also limited to BIhigh. A 10 seconds window is given if discharge rate is below BIcrit (e.g. for torque load, or to allow PWM controller to draw more power from PV array).
  • Overcharge protection
    Start beeping if battery is overcharged i.e. charged above BVfull or at a rate higher than BIhigh. We cannot control the PWM controller with Arduino. But we can cry.

Alarms

Raise alarm (start beeping) if any of the following is true:

  • Battery is above BVfull
  • Battery is below BVlow
  • Battery is being charged at a rate higher than BIhigh
  • Battery is being discharged at a rate higher than BIhigh
  • During daytime, if sunlight is sufficient:
    • Battery is being discharged (at any rate; we should be using solar)
    • Inverter turned off (and switched to grid; we should be using solar)
      • Exact reason of turning off the inverter is shown on screen. See warnings below.

Before raising alarms, give a 5-30 seconds window for temporary spikes / dips in battery voltage / current.

On-Screen Warnings

Inverter turned off (and switched to grid) due to:

  • E 1: Battery low – voltage below BVlow
  • E 2: Battery overloaded – high current (> BIhigh) drained
  • E 3: Load higher than the solar power (during daytime) – due to morning / evening, clouds etc.
  • E 4: Load higher than the solar power (during daytime) – your fault.

Display + Settings

  • Use push buttons to jump to the next screen (center) and to change the configurable values (left and right).
  • First 4 screens are informatory. On these screens, setting buttons turn off the display. Display also turns off if no activity is detected for 5 minutes. Press any button or wave hand (to trigger infrared obstacle detector) to turn the display back on.
  • When display is on, waving hand before IR sensor rotates the display between informatory screens.
  • On SAVE screen (last), setting buttons save the changed settings, if any. Then it jumps to the first screen. If it doesn't, there is a problem with the set values such as:
    • BVlow higher than BVfull
    • BVcrit higher than BVlow
    • BIhigh higher than BIcrit
    • BIlow higher than BIhigh
    • BIcrit-high window higher than BIhigh-low window
    • BIhigh-low window higher than BIbelow-low window
  • Display jumps back to the first screen after 30 seconds of inactivity on settings screens.

Hardware Components

  • Arduino Nano [1] – main controller for sensor reading and decision-making.
  • INA219 Bi-Direction DC Current Sensor Module [2] [3] + FL-2C Shunt Resistor 100A 75mV [4] [24] – used to measure battery voltage and current (charging and discharging).
    • Default I2C Address: 0x40
    • Recommended configuration is to sense high-side current i.e. shunt is connected to the positive terminal of battery, and negative terminal is shared with circuit GND.
    • Max bus voltage (across Vin- and GND): 0V – 26V (makes sense only in case of high-side sensing)
    • Max measurable current:
      • With on-board R100 (0.1 Ω) SMD shunt: 3.2 A
      • With 100A 75mV (= 0.00075 Ω) shunt: 436A (= 0.32767V / 0.00075Ω, theoretically)
    • When sensing high-side, bus voltage + shunt voltage gives the battery voltage.
      • Alternatively, use 0–25V DC Voltage Sensor Module [5]
  • 2 x HCPL-7840 Isolation Amplifier [23] + 2 x CA3130 Operational Amplifier [25] + FL-2C Shunt Resistor 100A 75mV [4] [24] – used to measure PV array voltage and current.
    • HCPL-7840:
      • Required because PWM charge controller does not have common GND across its input (PV array) and output (battery) sides.
      • Power supply: 4.5V-5.5V, current: 11mA
      • Input voltage:
        • Input is differential voltage i.e. 2 input pins.
        • Common-mode range: (input pins must stay between) –2.0V … VDD1 + 0.5 V ("Steady-State Input Voltage" [27])
          • E.g. up to ~5.5V on 5V supply
          • If pushed above or below the range (due to spikes), the internal clamp diodes are forward-biased and start dumping current. Use a series resistor like ~100kΩ to limit the current below ~10µA.
        • Differential range:
          • ±100mV with non-linearity ≤0.20%
          • ±200mV with non-linearity ≤0.35%
          • ±308mV absolute maximum
          • But we are limited to +75mV (or ±37.5mV) due to CA3130 input limit.
      • Input bias current: 0.5µA-2µA (40 MΩ differential, 20 MΩ common mode impedance)
      • Gain: 8V/V
      • Output voltage:
        • Common mode i.e. at 0V input, both outputs sit at 2.5V (with 5V supply). VOUT+ goes above 2.5V and VOUT- goes down by the same value. VOUT+ - VOUT- gives the amplified differential voltage.
        • Absolute output voltage range is 0.5V-4.5V.
        • Linear differential swing range is ±0.8V around mid-point i.e. 1.7V-3.3V (200mV input).
      • Output current: 15 mA
    • CA3130:
      • Used in differential configuration to amplify the small output of HCPL-7840 so that it covers a good Arduino ADC range (> 50%).
      • Power supply: 5V-16V (single), current: 10mA
      • Input voltage (common-mode range, across inverting and non-inverting terminals):
        • Not rail-to-rail. Upper limit is far below the supply voltage.
        • −0.5V to ~V+ − 2.2V (linear amplification) e.g. −0.5V to 2.8V for 5V supply.
        • ~V+ + 8V is the absolute maximum survival tolerance (non-linear).
      • Input bias current: 1mA
      • Gain: ≥50kV/V (≈94 dB)
      • Output voltage:
        • Not true rail-to-rail.
        • V- + 1.5V to V+ - 1.5V (with moderate load ~2kΩ)
        • V- + 10mV to V+ - 10mV (with light load ≥10MΩ)
        • Arduino GPIO pins input impedance is ~100MΩ, but is considered ~10kΩ due to a small sample-and-hold capacitor connected, when taking analog inputs. So expect up to a few hundred milli-volts offset.
      • Output current: 20 mA
      • Tie the Strobe pin 8 to GND to disable the output (output voltage jumps to V+).
      • CA3130 has non-R2R input and nearly R2R output (10mV to 1.5V headroom from each rail). For more precision use MCP602 / MCP6002.
    • See appendix 1 for alternate parts / approaches.
    • A more complex way to detect sunlight sufficiency is by using a BH1750 Light Sensor Module [8].
  • ZMPT101B AC Single Phase Voltage Sensor Module [6] – measures grid voltage. Alternatively, simply detect the presence of AC power using:
    • 220V AC to 5V DC Converter
    • AC 220V Optocoupler Isolation Module
    • H11AA1 Optocoupler [15]
      • Input LEDs forward voltage drop: 1.2-1.5V, forward current: 10mA (max 60mA).
  • DS3231 Precision RTC Module [7] – remembers time so that to schedule solar on / off time in the morning and evening respectively.
    • I2C Address: 0x68
  • Relay Module [9] (in NO mode) – used to switch AC load between grid and inverter output.
  • 5V Relay [10] (in NC mode) – used to turn off inverter when not needed (to save battery) or during battery protection events. This relay is installed inside the inverter in series with the main power button.
    • Max current: 60mA
  • 8-Digit Display MAX7219 Module [11] – show battery (dis)charge current / voltage / power, and configurable values.
  • 5-12V Active Buzzer [12] – for alarms.
    • Max current: 40mA (at 12V)
  • 3 x Push Buttons (momentary) [13]:
    • Left (DOWN): Decrease a configurable value (setting).
    • Center (MENU): Switch between screens and turn off the display.
    • Right (UP): Increase a configurable value (setting).
  • IR Obstacle Sensor [22]:
    • Wave hand to turn on the display, or rotate between informatory screens.
  • MP2307 Buck Converter [21] – converts 12V to 9V for Arduino Nano.
  • LM2596 Buck Converter [14] – converts 12V to 5V for other modules.

For Buzzer:
  • 2N3904 transistor – controls voltage supply (5-12V) to the buzzer to adjust the loudness.
    • Collector-emitter voltage: 40V
    • Max collector current: 200mA
    • Max base current: 20mA (200mA / 10)
    • Max base-emitter voltage drop: 0.95V (fully saturated)
    • Transition frequency: 300MHz | Rise/fall time: 35ns (switching speed matters for PWM)
    • Linearity:
      • Base voltage vs. collector current is exponential.
      • For linearity, use 2N3904 in closed-loop with an op-amp like LM358 / CA3130.
      • For even more accuracy, use a DAC like MCP4725 instead of filtered PWM.
  • 470 Ω resistor – between the transistor (2N3904) base and Arduino pin to protect high current draw.
    • For 100mA current draw from the transistor (5V - 0.7V / 10mA)
  • 1N4007 diode – for flyback protection with the buzzer.
  • Insert an RC filter (b/w Arduino and transistor, or b/w transistor and buzzer), or only a capacitor parallel to the buzzer, to flatten / smooth out the averaged output of PWM.
For Relay (inverter)
  • 2N3904 transistor – drives the relay.
  • 470 Ω resistor – between the transistor (2N3904) base and Arduino pin to protect high current draw.
    • For 100mA current draw from the transistor (5V - 0.7V / 10mA)
  • 1N4007 diode – for flyback protection at the relay coil.
  • 47µF 16V capacitor – for stabilizing voltage at the relay coil.
    • Purpose is to act as a local energy reservoir, supplying extra current to avoid the voltage dip during the initial surge when the relay coil is energized. But size it wisely (or add a small resistor in series) to avoid high inrush current to the capacitor itself when the transistor turns on (though BJTs can tolerate short over-current spikes).
  • 2 x 1N5822 Schottky diode – for diode ORing. Power sources isolation is required because inverter relay is triggered from 2 sources in parallel: Arduino and a power adapter. Latter is used to bypass the protection system.
Others:
  • 3 Pin T-Block Connector – AC mains connections:
    1. 220V live
    2. 220V neutral
    3. Output of Relay Module to switch AC load between grid and inverter output. It controls a high-power 220V-driven changeover relay. Latter can also be triggered from AC mains to bypass the protection system.
  • 4 x 2 Pin T-Block Connector for:
    1. 12V supply (battery)
    2. Relay (inverter)
    3. 2 x FL-2C Shunt Resistor

    • High-frequency electrical noise from the inverter may cause excessive RMS current in fuse, even though DC measurement appears low (multimeter shows average DC). This high RMS may cause localized heating in fuse due to skin effect. To avoid this:
      • Use a slow-blow fuse.
      • Or run dedicated pairs of wires (positive and negative) from the battery to the inverter and Arduino circuit.
      • Or at least use a long wire for Arduino circuit power supply.
  • DC Power Jack (male + female) – to make a detachable connection to Relay (inverter).
  • Veroboard Dotted (145mm x 65mm) – circuit designing.
  • 2.54mm Pitch Female Header – soldered with the veroboard to connect the sensor modules.
  • Plastic Enclosure Box (160mm x 96mm x 50mm) – enclose the circuit.
  • 4 x Plastic PCB Circuit Board Spacer – between veroboard and the box bottom.
  • 2-Core Cable – for 12V power supply, Relay (inverter) and FL-2C Shunt Resistor connections.
  • 3-Core Cable – for AC connections.

  • Shunt Resistor: Place between the battery positive terminal and the load.
  • INA219 Connections:
    • Vin+ → Shunt's battery-side terminal (charging would give negative current value)
    • Vin– → Shunt's load-side terminal (also connected to +ve terminals of inverter and solar charge controller)
    • GND → GND rail
    • VCC → 5V rail
    • SDA → Arduino Nano A4 (SDA)
    • SCL → Arduino Nano A5 (SCL)
  • Should remove the on-board SMD shunt? If 0.00075 Ω shunt is placed in parallel with the onboard 0.1 Ω SMD resistor, it becomes 0.0007444 Ω. At 50A:
    • Voltage drop: 37.2 mV
    • Current and power dissipation are within limits (so no need to remove the on-board SMD shunt):
      • External shunt: 49.6 A (1.85 W dissipation)
      • Internal shunt: 372 mA (13.8 mW dissipation)

2. HCPL-7840 + CA3130 + FL-2C Shunt (PV Array Voltage + Current Sensor)

LM7805 Power Supply:
  • Connections:
    • IN →
      • 150Ω → PV+ (shares power dissipation with LM7805 IC)
      • 15V Zener Diode (1N4744A) [Anode] → PV-
    • GND → PV-
    • OUT → HCPL-7840 power input
  • See appendix 2 for resistor + diode evaluation.
Voltage Sensing:
  • Voltage Divider:
    • PV+ → 47kΩ → Junction → 150Ω → PV-
    • Drops 25V (VOC) to ~75mV for HCPL-7840 input.
    • 150kΩ:470Ω draws/dissipates less power but:
      • Is more susceptible to EMI / cable noise due to high impedance.
      • May induce input-bias errors in the HCPL-7840 due to ultra low current draw (~0.17mA).
  • HCPL-7840 Isolation Amplifier:
    • Input:
      • VDD1 → LM7805 5V
      • GND1 → LM7805 GND (PV-)
      • VIN+ →
        • 33 Ω → Voltage divider junction
        • Switching diode (1N4148) [Anode] → LM7805 GND (PV-)
      • VIN- →
        • 33 Ω → LM7805 GND (PV-)
        • Switching diode (1N4148) [Anode] → LM7805 GND (PV-)
    • Output:
      • VDD2 → Arduino-side 5V rail
      • GND2 → Arduino-side GND rail
      • VOUT+ → 1 kΩ resistor (Rg) → CA3130 NON-INV INPUT
      • VOUT- → 1 kΩ resistor (Rg) → CA3130 INV INPUT
  • CA3130 Operational Amplifier:
    • V+ (Pin 7) → 5V rail
    • V- (Pin 4) → GND rail
    • INV INPUT (Pin 3) →
      • 6.8 kΩ resistor (Rf) → Self OUTPUT (feedback)
      • HCPL-7840 output
    • NON-INV INPUT (Pin 2) →
      • 6.8 kΩ resistor (Rf) → GND rail
      • HCPL-7840 output
    • OUTPUT (Pin 6) →
      • 100Ω → Arduino A6
      • 5.1V Zener Diode (1N4733A) [Anode] → GND rail
    • Offset Nulling:
      • OFFSET NULL (Pin 1) → 100kΩ potentiometer → OFFSET NULL (Pin 5)
      • Potentiometer wiper → GND rail
  • Calibration:
    • Divider output at 25V PV input: 25V × 150Ω ÷ (150Ω + 47kΩ) = ~75mV
    • HCPL-7840 output: 75mV × 8 = 2.8V - 2.2V = 600mV
    • CA3130 output: 600mV × 6.8kΩ ÷ 1kΩ = 4.1V
Current Sensing:
  • Shunt Resistor: Place in series with the charge controller negative input.
    • Due to common-mode range limit of HCPL-7840, we cannot sense high-side.
  • HCPL-7840 Isolation Amplifier:
    • VIN+ is connected to shunt controller-side, VIN- to PV-side. Use same resistors and diodes.
  • CA3130 Operational Amplifier:
    • Use 10 kΩ resistors for Rf.
    • Use Arduino A7 for input.
  • Remaining details are same as that of voltage sensor.
  • Calibration:
    • Shunt output at 60A PV input: 75mV × 60A ÷ 100A = 45mV
    • HCPL-7840 output: 45mV × 8 = 2.68V - 2.32V = 360mV
    • CA3130 output: 360mV × 10kΩ ÷ 1kΩ = 3.6V
Filtering:
  • Add 0.1µF Ceramic Capacitor (high frequency decoupling) + 10µF Electrolytic Capacitor (stability) to all power inputs and outputs (LM7805, HCPL-7840, CA3130).
  • Add 10nF Ceramic Capacitor across HCPL-7840 IN+ and IN–.

Appendix 1: HCPL-7840 + CA3130 Alternatives (Isolated PV Current / Voltage Sensing)

  • HCPL-7520 is slightly less performant but has a single (non-differential) output.
  • Even LM358 can replace CA3130 but it's not rail-to-rail.
  • An isolated analog/linear amplifier with higher or adjustable gain, like AMC3302 (x41) / Si8921AD (x33), so that 2nd stage op-amp is not needed.
  • A linear opto-isolator like IL300 / HCNR200 / LOC110 / ISO122 (precision amplifier with unity gain). PC817, 4N35 (better linearity) etc. are digital/switching types, highly non-linear; not ideal for analog signals.
    HCNR200 and ISO122 are particularly good for PV voltage measurement as they can straight pass 0-5V input as output with isolation.
  • V/F converter like LM331 + opto-isolator like 6N137 / HCPL2630 (dual-channel). This overcomes the non-linearity problem as we pass frequency instead of voltage. Use an instrumentation amplifier like INA818 to amplify the shunt voltage. A basic op-amp like LM358 won't work precisely for amplifying small voltages.
  • Flying capacitor method. Charge a capacitor on PV side, and discharge on Arduino side to measure the voltage.
  • Use a galvanically-isolated sensor, like ACS758 (for current only), which is a hall-effect based linear current sensor. LV 25-P is a similar one for voltage sensing.
  • Instead of analog isolation, do digital signal isolation:
    • Op-amp buffer (voltage stabilization): A rail-to-rail, zero-drift op-amp like OPA333AIDCKR
    • Current sensing: Shunt + INA219 or op-amp buffer
    • Voltage sensing: Voltage divider + op-amp buffer
    • Current / voltage sensor +
      • ADC like ADS1115 + I2C isolator like ADuM1250 / ISO1540
        • I2C isolation using opto-couplers is complicated due to bidirectional communication (SDA, SCL).
      • ADC like MCP3008 / MCP3208 + 4 x 6N137 for SPI isolation
      • MCU (I2C to UART or PWM) + 6N137 (opto-isolation)

Appendix 2: LM7805 Resistor + Zener Diode Evaluation

  • Calculations:
    • VPV: 25V
    • V1N4744A: 15V
    • VR: 25V - 15V = 10V
    • VLM7805: 15V - 5V = 10V
    • IR: 10V / 150Ω = 66.7mA
    • ILM7805: ~40mA (35mA for 2 x HCPL-7840 + 5mA for LM7805 quiescent)
    • I1N4744A: 66.7mA - 40mA = 26.7mA
    • PR: 10V × 66.7mA = 0.67W (≥1W resistor)
    • PLM7805: 10V × 40mA = 0.4W (acceptable)
    • P1N4744A: 15V × 26.7mA = 0.4W (acceptable)
    • VRmin: 40mA × 150Ω = 6V (when 1N4744A is unclamped)
    • VPVmin: 7V + 6V = 13V (LM7805 won't work below this)
  • Bigger resistor size requires high PV voltage (we need at least 7V after resistor drop).
  • Smaller resistor size dissipates more heat due to high current going through Zener diode.
  • We need a balance. A 100Ω 1/4W resistor with no Zener diode should also work:
    • 0.16W loss at resistor, 0.64W at LM7805. 45°C (ambient) + 32°C (loss) = 72°C is not good, though acceptable.
    • Protection against voltage spikes (e.g. due to PWM switching transients, lightning induction) is lost.
    • Power dissipation is reduced from 1.47W to 0.8W.
  • Better alternatives: MP2307 buck converter, or B0505S isolated DC/DC converter.
